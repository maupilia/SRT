ACS Command Center¶

Fig.1 The ACS monitor shows the state of the containers related to each DISCOS component.

Active Surface¶

Fig.2 This monitor shows the status of the actuators in a graphical representation of the Active Surface and its configuration.

Logging Display¶

Fig.3 The Logging Display shows the log messages related to the observation. New messages are shown on top of the previous ones.

Meteo Client¶

Fig.4 The Meteo Client window shows the atmospheric temperature and the wind parameters (including wind direction) using a graphic interface.

Nuraghe-mng¶

Fig.5 Nuraghe-mng is the destination for your schedules, and is the machine where you run the system and where you should find the input terminal and all the monitors.

OperatorInput¶

Fig.6 In the input console the users can write Nuraghe commands. The prompt is just a sequential number enclosed in <>. If a command is properly read, the system replies repeating the command itself, followed by the operation results (if they are foreseen).

AntennaBoss¶

Fig.7 The AntennaBoss monitor shows the target info, indicating the commanded and actual positions pointed by the antenna. It also gives a feedback on the pointing accuracy and on the overall antenna status.

GenericBackend¶

Fig.8 The monitor GenericBackend shows the backend setup parameters related to each section.

The monitor GenericBackend in the K-band configuration.

Mount¶

Fig.10 Observers need to focus only on the Mount status (that you can see indicated by the green box) and on the actual position of the axis expressed in Azimuth and Elevation (shown in blue), compared to the commanded positions (which are referred to under the actual positions with the label “Cmd Az.” and “Cmd El.”).

Fig.11 The Mount monitor after the park of the antenna by using > antennaPark

Observatory¶

Fig.12 The Observatory monitor shows the station coordinates and times.

ReceiversBoss¶

Fig.13 The ReceiverBoss monitor summarizes the frontend setup parameters. The bottom part is devoted to the derotator (dewar positioner), when available.

The ReceiverBoss monitor in the L-band configuration.

The ReceiverBoss monitor in the K-band configuration.

Scheduler¶

Fig.16 The Scheduler monitor shows details on the selected data acquisition devices and on the running schedule, if any.

Fig.17 The Scheduler monitor after the interruption of the current subscan by using > stopSchedule or > haltSchedule

MinorServo¶

Fig.18 The MinorServo monitor shows the current setup code and the minor servo status and movement. In this case the image refered to the C-band configuration.

Fig.19 The MinorServo monitor in the L-band configuration.

Fig.20 The MinorServo monitor in the K-band configuration.

Fig.21 The MinorServo monitor after the park of the minor servos by using > servoPark

Calibration tool client¶

Fig.22 In the Calibration tool client window the subscan currently being acquired is shown in real-time (upper plot), even if in a low-resoltution. In the lower plot, the last completed subscan - in its full sampling - is shown. We can read the information about the pointing of focus offsets (peakoffsets), the beam size (HPBW), etc.

Fig.23 Calibration tool client window related to a focus subscan.

GenericBackendX¶

Fig.24 A second GenericBackend panel shows the setup parameters of each section of Xarcos.
